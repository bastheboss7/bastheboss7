## ğŸš€ How to Contribute

### For Small Changes (typos, minor edits):
1. Click the "Edit" button (pencil icon) on any file
2. Make your changes
3. Submit a pull request with a clear description

### For Larger Contributions:
1. **Fork** the repository
2. **Create** a new branch: `git checkout -b feature/your-framework-name`
3. **Make** your changes following the template structure below
4. **Commit** with clear messages: `git commit -m "Add: Incident Response Playbook"`
5. **Push** to your fork: `git push origin feature/your-framework-name`
6. **Submit** a pull request with:
   - Clear title
   - Description of what you're adding
   - Why it's valuable
   - Any context needed

## ğŸ“ Content Guidelines

### Template Structure
Each document should include:

```markdown
# [Framework/Template Name]

## Purpose
*Why this exists and what problem it solves*

## When to Use
*Specific scenarios where this applies*

## How to Use
*Step-by-step implementation guide*

## Template/Framework
*The actual artifact*

## Customization Guide
*How to adapt for different contexts*

## Real-World Example
*Optional: Anonymous case study*

## Related Resources
*Links to complementary frameworks*
```

### Quality Standards
- **Practical over theoretical:** Real-world applicable
- **Clear and concise:** No jargon without explanation
- **Well-formatted:** Use headings, bullets, tables
- **Examples included:** Show, don't just tell
- **Tested in production:** Battle-tested > hypothetical
